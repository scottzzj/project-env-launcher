项目页面要解决的问题

项目路径可配置

换电脑后不依赖硬编码路径。通过项目页面重新选择目录，后续模块识别、配置读取和 Maven 启动都基于这个路径。

模块来自真实扫描

项目添加后，系统扫描 Maven 结构和配置文件，识别可启动模块以及端口信息，不在页面中放固定业务模块。

操作闭环完整

新建、编辑、删除、刷新识别都在页面内完成；删除使用独立确认框，不调用浏览器原生弹窗。

和其他页面的关系

  • 项目管理保存项目名称和本机目录。
  • 模块管理读取项目目录，识别模块和启动端口。
  • 配置管理按“环境 + 模块”保存常改配置。
  • 监控面板基于项目、环境和模块执行 Maven 启停。

数据边界

预览页包含通用 mock 项目，用来展示项目管理页面布局；真实运行时，项目、环境、模块和配置全部来自本机 SQLite 与实际项目扫描。数据库、日志、临时配置文件已排除在代码提交之外。
