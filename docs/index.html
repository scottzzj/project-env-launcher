Project Env Launcher

本地项目环境启动与配置管理工具

用一个本地 Web 控制台管理项目路径、环境、模块端口和常用配置，并通过 Maven 启动指定模块。运行数据来自本机 SQLite 与实际进程扫描，预览页不包含任何本地业务配置。

查看源码 本地启动 
npm install
npm run dev:all

# 或构建后运行
npm run build
npm start

功能预览

项目管理

手工维护本机项目路径，支持编辑、删除和刷新识别结果，路径变更后可在页面中重新配置。

环境与配置

维护环境清单，并按“环境 + 模块”保存 Nacos、RabbitMQ、Redis、数据库与启动端口等常改项。

模块启动

从项目中识别可启动模块，按当前环境批量启动或关停，启动过程保留在当前页面的实时日志中。

本地工作流

1. 添加项目选择本机项目目录。
2. 维护环境录入实际使用的环境。
3. 识别模块扫描 Maven 模块与端口。
4. 保存配置按环境和模块落库。
5. 启停服务执行 Maven 并查看日志。

数据边界

项目运行时会在本地生成 SQLite 数据库、启动日志和临时配置文件；这些内容已通过 .gitignore 排除，不会进入公开代码提交。换电脑后重新配置项目路径和环境即可使用。
